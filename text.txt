গিট ইনস্টল করার পর  কম্পিউটারে মাত্র একবারই এই ২টি গ্লোবাল কমান্ড দিতে হয়, যাতে গিটহাব বুঝতে পারে কোডটি আসলে কে সাবমিট করছে
---------------------------------------------------
--------------------------------------------------
git config --global user.name "Your Name"
---------------------------------------------------
--------------------------------------------------
git config --global user.email "your.email@example.com"
---------------------------------------------------
--------------------------------------------------
git init :  গিট প্রজেক্ট শুরু করা  এই কমান্ডটি আপনার লোকাল কম্পিউটারের ফোল্ডারটির ভেতরে একটি খালি গিট রেপোজিটরি (Repository) তৈরি করে। সহজ কথায়, গিটকে বলা যে—"এখন থেকে এই ফোল্ডারের ফাইলগুলোর ওপর নজর রাখো।"
---------------------------------------------------
--------------------------------------------------
git add README.md : নির্দিষ্ট ফাইল গিটের ট্র্যাকিংয়ে নেওয়া এই কমান্ডটি শুধু README.md নামের ফাইলটিকে পরবর্তী সেভ বা কমিট করার জন্য রেডি (Stage) করে। (নোট: আপনি যদি সব ফাইল একসাথে রেডি করতে চান, তবে git add . ব্যবহার করতে হয়।)
---------------------------------------------------
--------------------------------------------------
git commit -m "first commit" : ফাইলগুলো মেসেজসহ সেভ করা: এর মাধ্যমে আপনার রেডি করা ফাইল বা পরিবর্তনগুলোকে একটি নির্দিষ্ট মেসেজ (যেমন: "first commit") দিয়ে লোকাল কম্পিউটারে পাকাপাকিভাবে সেভ করে রাখা হয়।
---------------------------------------------------
--------------------------------------------------
git branch -M main : প্রধান ব্রাঞ্চের নাম ঠিক করা: এই কমান্ডটি আপনার প্রজেক্টের মেইন বা মূল ব্রাঞ্চের (Branch) নাম পরিবর্তন করে main করে দেয়। (আগে ডিফল্ট নাম master থাকত, এখন স্ট্যান্ডার্ড নিয়ম হলো main রাখা)।
---------------------------------------------------
--------------------------------------------------
git remote add origin [location] : গিটহাবের সাথে সংযোগ করা: [location] এর জায়গায় আপনার গিটহাবের খালি প্রজেক্টের লিঙ্কটি বসে। এই কমান্ডটি আপনার কম্পিউটারের ফোল্ডারটির সাথে গিটহাবের অনলাইনের ফোল্ডারটির একটি ভার্চুয়াল কানেকশন বা লিংক তৈরি করে দেয়।
---------------------------------------------------
--------------------------------------------------
git push -u origin main : কোড অনলাইনে পাঠানো: এই কমান্ডটি আপনার কম্পিউটারে সেভ থাকা সব কোড বা ফাইল ধাক্কা দিয়ে (Push) গিটহাবের সার্ভারে আপলোড করে দেয়। -u দেওয়ার কারণে পরের বার থেকে আর পুরো কমান্ড লিখতে হয় না, শুধু git push লিখলেই কোড চলে যায়।
---------------------------------------------------
